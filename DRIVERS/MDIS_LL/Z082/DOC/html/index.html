Z82 MDIS Driver Documentation

This is the documentation of the MDIS low-level driver for the 16Z082 impulse counter IP core.

The 16Z082_IMPULSE module can be used to count the impulses from e.g. wheel rotation sensors. To be able to support a wide range of wheel speeds the debouncer is programmable.


Variants

(no variants)


Functional Description


General

The IP core acts as an impulse counter with programmable debounce capability to adapt rotation sensors etc. to their signal source. The possible debounce values are described below.


Reading and writing, Get- and SetStats


Z82_CLR_RESULT SetStat, clears the RESULT register to 0
This should be done as initial application step.


Z82_GET_RESULT GetStat, gets the current value of RESULT register.
A count value represents one pulse of the input.


Z82_GET_OVF GetStat, gets the RESULT registers overflow frag.
The overflow flag should be read after the RESULT register
or if the RESULT value increases near the unsigned 32-bit
value limit (2^32 or 4294967296 impulses).


Z82_DEBOUNCE Get/SetStat, gets/sets the debounce register value. The
debounce time is a value of 0-1023.


Interrupt handling and signals


The driver supports no interrupts as the IP core is a counter only
